IMInterstitialDelegate Protocol Reference

Conforms to NSObject
Declared in IMInterstitialDelegate.h

Overview

Delegate to receive state change message from IMInterstitial.

Tasks

  • – interstitialDidReceiveAd:

    Sent when an interstitial ad request succeeded.
  • – interstitial:didFailToReceiveAdWithError:

    Sent when an interstitial ad request failed
  • – interstitialWillPresentScreen:

    Sent just before presenting an interstitial. After this method finishes the interstitial will animate onto the screen. Use this opportunity to stop animations and save the state of your application in case the user leaves while the interstitial is on screen (e.g. to visit the App Store from a link on the interstitial).
  • – interstitialWillDismissScreen:

    Sent before the interstitial is to be animated off the screen.
  • – interstitialDidDismissScreen:

    Sent just after dismissing an interstitial and it has animated off the screen.
  • – interstitialWillLeaveApplication:

    Callback sent just before the application goes into the background because the user clicked on a link in the ad that will launch another application (such as the App Store). The normal UIApplicationDelegate methods like applicationDidEnterBackground: will immediately be called after this.
  • – interstitialDidInteract:withParams:

    Called when the interstitial is tapped or interacted with by the user Optional data is available to publishers to act on when using monetization platform to render promotional ads.
  • – interstitial:didFailToPresentScreenWithError:

    Called when the interstitial failed to display. This should normally occur if the state != kIMInterstitialStateReady.
